Lists are common in HTML. For example, a NAV bar can be a list. They are styled using CSS sometimes. It is important to know our skeleton.

Ordered List - These elements in this list might be numbered, such as steps in a recipie. The order is important here.

Unordered List - These elements are for things that the order does not matter, such as a shopping list, or ingredients list.

Ordered List - OL

OL Stands for ordered list. List items are indicated by li. The items here are automatically numbered. as long as elements are added inbetween the OL, the items will be counted.

  1. Add Flour
  2. Mix in salt
  3. Mix in eggs.

Unordered List - UL

If the order doesn't matter, we use a ul.

  • Eggs
  • Milk
  • Salt
  • Flour

Comments

Users can add comments to their code. When you learn a new feature, you can add a comment to say what it does. Comments are written as follows < !-- comment -->. In this case, it appears since I left a space there to see, but it will be removed to add in comments.

Nested Lists

Lists can be nested. For example, for a navi bar.

This may be a list with a sublist.

  • Mix eggs with whisk
    • Fold in mixture
    • Wait 5 minutes

Can be used for lists with sublists, but can be difficult to keep track of the list orders. Comments can be used to mark the beginning and the ends of lists and sublists.
